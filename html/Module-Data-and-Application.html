Hackathon

1 - Présentation

Dans le cadre de la formation ISS, j'ai pu participer à un hackathon en équipe de 4 étudiants entre septembre et novembre 2021. En effet, c'est un concours international sur le domaine de l'IoT organisé en Corée du Sud par le KETI, l'ETSI et le comité de normalisation OM2M.

2 - Partie technique

Le but était de développer une idée de projet innovant autour des réseaux de capteurs et de développer une prototype du système inventé. Pour cela, chaque équipe avait un kit de développement comprenant des capteurs ainsi qu'une carte ESP8266.

Les paramètres environnementaux dans une maison sont cruciaux pour préserver la santé des habitants. Le but de ce projet est de surveiller en temps réel les gaz toxiques et le niveau de luminosité à distance et de les afficher à l'utilisateur via une application Web. Nous avons utilisé des composants matériels simples et largement répandus pour rendre le projet facilement reproductible.

Web sémantique et traitement des données

1 - Présentation

Le module de Web Sémantique était composé de deux cours théoriques, au cours desquels il nous a été présenté les notions de base de ce domaine et notamment la notion d’ontologie. Le Web sémantique est un standard pour donner du sens aux données. J'ai ainsi pu développer les compétences suivantes :

  • Comprendre un modèle de web sémantique
  • Créer, enrichir et utiliser une ontologie
  • Utiliser le logiciel « Protégé ».

2 - Partie technique

Le Web sémantique est un standard pour donner du sens aux données. À travers une série d'exercices, nous avons défini les objets, les propriétés entre eux et les propriétés liées aux données. En organisant les informations comme telles, les machines peuvent les utiliser pour tirer des conclusions. Nous avons utilisé Protégé pour spécifier nos ontologies (modèle) et HermiT pour les inférences.

Grâce à cette UF, j’ai ainsi pu avoir un premier contact avec le web sémantique. Le bon déroulement des TP m’a permis d’acquérir les compétences attendues et de les mettre en application dans un cas concret. La puissance de déduction du raisonneur de Protégé laisse également entrevoir les bases de l’intelligence artificielle, domaine en plein développement aujourd’hui.

Analyse et Traitement de données : Big Data

1 - Présentation

L'enseignement "Analyse et Traitement de données" est décomposé en une séquence de cours et une séquence de TP.

Le but de ce cours est mettre en oeuvre et de comparer différents algorithmes de clustering tout d’abord à partir de quelques méthodes fournies par scikit-learn.

  • k-Means
  • clustering hiérarchique (agglomératif)
  • DBSCAN

2 - Partie technique

L'objectif de ces TPs de mettre en oeuvre et comparer différents algorithmes de clustering. Pour ce faire, nous nous sommes basés sur les méthodes fournies par le package python scikit-learn.

Dans un premier temps, nous avons réaliser une comparaison détaillée des méthodes de clustering suivantes : k-means, clustering agglomératif et DBSCAN. L’objectif étant d’identifié les points faibles et forts de chaque méthodes. Pour ce faire, nous avons étudier, entre autres, les résultats et les performances de chaque méthode.

Dans un second temps, Dans cette partie, nous avons appliqués les différentes méthodes de clustering sur de nouveaux jeux de données. Les données correspondent à des données artificielles de petites dimensions et à un jeu de données "réel" ayant un nombre d’attributs plus important.

Grâce à cette UF, j’ai ainsi pu avoir un premier contact avec le Big Data. Le Big Data est, de nos jours, un domaine très prisé, utilisé dans de nombreux secteurs différents (finance, marketing, ingénierie…). J’ai donc apprécié d’avoir un module d’introduction sur ce sujet. J’ai ainsi pu prendre conscience de la complexité de la gestion d’un très grand nombre de données. Le côté très pratique de ce module, avec les TPs, m’a permis de comprendre comment y faire face, notamment grâce à l’utilisation de packages dédiées à cela dans le langage Python.

Ingénierie logicielle

1 - Présentation

Ce cours s'inscrit dans le projet de gestion automatique des salles d’un bâtiment du cours d'architecture de service présent dans l'UF "Middleware and Service".

2 - Partie technique

Il nous a été demandé d’organiser notre travail selon la méthode agile Scrum en s’appuyant sur l’outil Jira, notamment pour gérer l’organisation des sprints. Nous avons décomposé notre projet en questions et en histoires d'utilisateurs (user stories). Nous avons ensuite planifié des sprints pour atteindre nos objectifs dans un temps donné. Le fait d'avoir tout planifié et d'avoir des objectifs atteignables a facilité et simplifié notre travail.

Pour explorer ce qu'est l'intégration continue, nous avons effectué une série de travaux en utilisant Jenkins. Nous avons appris à construire du code avec lui, à mettre en place des tests, à le connecter à git pour le versionnement et à déployer une application.

Durant ce module, le déroulement étape par étape de chaque aspect a été bénéfique et m’a permis d’aborder le projet final avec toutes les cartes en main. J'ai pu acquérir des compétences en gestion de projet ce qui par la suite sera bénéfique pour ma carrière professionnelle.
